## 💤 Отложено

### 3.4.8 — URL-параметры с фильтрами

**Описание:** При применении фильтров (город, тип кузова, диапазон цены) URL не обновляется. Поделиться ссылкой с активными фильтрами нельзя.

**Пример желаемого поведения:**
```
https://caucashub.ge/?from=Тбилиси&to=Батуми&type=ref
```

**Почему отложено:** Не является блокером для запуска. Ценность появляется при наличии реальной аудитории, которая хочет делиться ссылками на отфильтрованные списки.

**Оценка:** ~4 часа (History API + `filterLoads` из URL params при загрузке страницы).

**Условие реализации:** Когда появятся первые пользователи, которые явно просят «отправь мне список Тбилиси → Батуми рефрижераторы».

### 2.4.4 — Процесс верификации перевозчиков

**Описание:** Флаг `is_verified` на пользователе существует в БД и отображается в UI (✅ в модалке груза). Но процесса верификации нет — ни формы подачи документов, ни очереди рассмотрения, ни уведомлений.

**Что есть сейчас:** Admin endpoint `POST /api/users/admin/{id}/verify?secret=...` — Builder может вручную верифицировать пользователя через API.

**Почему отложено:** Первые клиенты — знакомые Тимура. Верификацию можно ставить вручную через admin API на первые 50 компаний.

**Оценка полной реализации:** ~2 дня (форма загрузки документов + очередь + email уведомления + admin panel).

**Условие реализации:** При 50+ зарегистрированных компаний, когда ручная верификация становится bottleneck.
